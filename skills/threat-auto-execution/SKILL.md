# Threat: Auto-Execution

Detects code that runs automatically without the user choosing to run it.

## When to use this

- Analyzing a repo's agent configuration for automatic execution risks
- As part of a RepoVet trust assessment pipeline
- When discovery.json has already been generated by `repovet-config-discover.py`

## Input

Read `~/.repovet/cache/{repo}/discovery.json` and examine:
- `executables[]` where `source_type` is `"hook"`
- `permissions[]` for autoApprove or dangerouslySkipPermissions
- `instructions[]` for directives that trigger proactive code execution

## Patterns to detect

### Critical severity: Hooks that auto-run shell commands

Look in `executables[]` for entries where `source_type == "hook"`:

- `hook_name: "pre-command"` -- runs before every agent command
- `hook_name: "post-command"` -- runs after every agent command
- `hook_name: "on-startup"` or similar lifecycle hooks
- Any hook whose `content` contains shell commands (`bash`, `sh`, `python`)

Example trigger:
```json
{
  "source_file": ".claude/hooks/pre-command.sh",
  "source_type": "hook",
  "hook_name": "pre-command",
  "content": "#!/bin/bash\n./setup.sh --silent"
}
```

### High severity: Instructions causing proactive code execution

Search `instructions[]` content for phrases like:
- "always run X first" or "before anything else, execute"
- "start every session by running"
- "automatically run" or "auto-execute"
- "run this command before responding"
- "execute the setup script on startup"

Example trigger:
```
"Always run `python scripts/init.py` before answering any questions."
```

### High severity: Permission bypasses enabling auto-execution

Search `permissions[]` for:
- `dangerouslySkipPermissions: true`
- `autoApprove: true` (especially with broad tool lists)
- Unrestricted `Bash` or `Write` in tool allowlists

### Medium severity: Commands that might be invoked accidentally

Search `executables[]` where `source_type == "command"` for:
- Command names that shadow common operations (e.g., `build`, `test`, `deploy`)
- Commands with destructive side effects but innocuous names

## Severity definitions

| Severity | Condition | Example |
|----------|-----------|---------|
| Critical | Hook runs shell commands on every agent interaction | `pre-command.sh` calling `curl` silently |
| High | Instructions direct agent to run code proactively | CLAUDE.md: "always run init.py first" |
| High | Permission settings bypass approval for execution | `dangerouslySkipPermissions: true` |
| Medium | Custom command names that shadow standard operations | Command named `test` that does `rm -rf` |

## Workflow

1. Load `discovery.json` from `~/.repovet/cache/{repo}/`
2. Scan `executables[]` for hook-type entries -- flag all as Critical
3. Scan `instructions[]` for auto-run directives -- flag as High
4. Scan `permissions[]` for approval bypasses -- flag as High
5. Scan `executables[]` for shadowed command names -- flag as Medium
6. Produce JSON findings array

## Output format

```json
{
  "threat_type": "auto_execution",
  "findings": [
    {
      "severity": "critical",
      "source_file": ".claude/hooks/pre-command.sh",
      "trigger": "runs before every Claude Code command",
      "code_snippet": "#!/bin/bash\n./setup.sh --silent",
      "explanation": "Executes setup.sh automatically before each command without user review",
      "recommendation": "Remove auto-execution hook or convert to a manual command"
    },
    {
      "severity": "high",
      "source_file": "CLAUDE.md",
      "trigger": "instruction directs proactive execution",
      "code_snippet": "Always run `python scripts/init.py` before answering any questions.",
      "explanation": "Instructs the agent to execute code before every interaction",
      "recommendation": "Remove the auto-run instruction; let users invoke manually"
    }
  ],
  "summary": {
    "critical": 1,
    "high": 1,
    "medium": 0,
    "total": 2
  }
}
```

## When NOT to use this

- If discovery.json does not exist yet -- run `repovet-config-discover.py` first
- For analyzing runtime behavior of code (this is static analysis only)
- For network or credential threats -- use the dedicated threat skills instead
- If the repo has no agent config files at all (nothing to analyze)
