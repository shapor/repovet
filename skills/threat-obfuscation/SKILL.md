# Threat: Obfuscation

Detects code that deliberately hides what it actually does.

## When to use this

- Analyzing a repo's agent configuration for hidden or disguised behavior
- As part of a RepoVet trust assessment pipeline
- When discovery.json has already been generated by `repovet-config-discover.py`

**Important context**: Obfuscation in agent config files is almost never legitimate.
There is no valid reason to base64-encode a shell command in a hook or CLAUDE.md.
All findings from this skill should be treated as High or Critical severity.

## Input

Read `~/.repovet/cache/{repo}/discovery.json` and examine:
- `executables[]` content for encoding, eval, and obfuscation patterns
- `instructions[]` for encoded or deliberately unclear directives

## Patterns to detect

### Critical severity: Encoded commands piped to shell

Decoding hidden payloads and executing them:
- `echo "Y3VybC..." | base64 -d | bash`
- `echo "68747470..." | xxd -r -p | sh`
- `python -c "import base64; exec(base64.b64decode('...'))"`
- `echo "H4sIAA..." | base64 -d | gunzip | sh`
- `printf '\x63\x75\x72\x6c' ...` (hex-encoded curl)

### Critical severity: Eval of constructed or encoded strings

Building commands dynamically to avoid static detection:
- `eval "$(echo Y3VybA== | base64 -d)"`
- `eval "$CMD"` where CMD was built from obfuscated parts
- `bash -c "$(cat /tmp/hidden_cmd)"`
- Python: `exec(decode_something(...))`, `eval(compile(...))`
- JS: `eval(atob('...'))`, `new Function(atob('...'))()`

### High severity: String concatenation to hide commands

Splitting suspicious commands to avoid pattern matching:
- `c="cur"; c+="l "; c+="http"; eval "$c"`
- `cmd = 'cu' + 'rl ' + 'ht' + 'tp'`
- Using variables to build command names character by character

### High severity: Character escaping tricks

Unicode escapes, quote insertion, or backslash tricks:
- `c\u0075rl` (unicode escape in curl)
- `cu''rl` or `cu""rl` (empty quotes to break pattern matching)
- `c\url` (backslash insertion)
- `$'\x63\x75\x72\x6c'` (bash hex escapes)

### High severity: Misleading comments or names

Code where comments do not match behavior:
- Comment says "check version" but code sends data to external URL
- Function named `cleanup()` that actually exfiltrates files
- Variable named `config` that contains an encoded payload
- README or CLAUDE.md describes benign behavior that differs from actual hook code

### Patterns to search for in executable content

```
# Base64/hex decode + execute
base64\s+(-d|--decode)\s*\|\s*(bash|sh|python|eval)
base64\.b64decode|atob\s*\(
xxd\s+-r\s+-p\s*\|
printf\s+.*\\x[0-9a-fA-F]{2}
# Compressed payloads
(gunzip|zcat|bunzip2|gzip\s+-d)\s*\|\s*(bash|sh)
# Eval (in agent config, almost always suspicious)
\beval\b\s|\bexec\b\s*\(|new\s+Function\s*\(
# String building + quote/escape tricks
\+\=.*\+\=.*eval|cmd\s*=.*\+.*\+
[a-z]''[a-z]|[a-z]""[a-z]|\\u00[0-9a-f]{2}|\$'\\x[0-9a-f]{2}
```

## Severity definitions

| Severity | Condition | Example |
|----------|-----------|---------|
| Critical | Encoded payload decoded and executed | `echo "..." \| base64 -d \| bash` |
| Critical | Eval of dynamically constructed command | `eval "$(echo ... \| base64 -d)"` |
| High | String concatenation to build commands | `c="cur"; c+="l"; eval "$c"` |
| High | Character escaping to avoid detection | `cu''rl`, `$'\x63\x75\x72\x6c'` |
| High | Comments that misrepresent code behavior | Comment: "ping server" / Code: sends files |

## Workflow

1. Load `discovery.json` from `~/.repovet/cache/{repo}/`
2. For each entry in `executables[]`, scan for base64/hex decode patterns
3. Check for eval/exec usage (almost always suspicious in agent config)
4. Look for string-building patterns that construct commands
5. Check for character escaping tricks (unicode, quotes, backslashes)
6. Compare comments/names against actual code behavior where possible
7. Assign severity (default High or Critical for all obfuscation)

```json

    "medium": 0,

```

## When NOT to use this

- If discovery.json does not exist yet -- run `repovet-config-discover.py` first
- For base64 encoding of data payloads (e.g., encoding JSON for an API body) -- that is normal
- For minified JS or compiled assets in the repo source (not agent config)
- For legitimate use of `eval` in the repo's application code (only flag in agent config)
- Environment variable expansion like `$HOME` or `$USER` is not obfuscation
